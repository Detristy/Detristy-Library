# 1. 定义
>[!definition] 定义
> 我们定义一个子序列 $L$ 是上升子序列当且仅当 $L$ 满足子序列中的任意两个数 $i,j$ 满足 $i<j$ ，其在原序列中的下标 $\mathrm{idx}$ 满足 $\mathrm{idx}_{i}<\mathrm{idx}_{j}$ ，保证关系是严格的。

# 2. 求解
## 2.1 状态定义和状态转移
通过上面的定义我们不难得到，上升子序列中的任意两个元素并不需要满足它们的下标一定相邻，只需要满足对应的相对关系即可。

接下来我们考虑如何在一个序列里找到最长的上升子序列，使用DP。

考虑设状态 $dp[i]$ 为底 $i$ 个数字结尾的最长上升子序列的长度，接下来我们看看状态怎么转移：

不妨考虑位置 $i$ 以前所有的位置 $j$ , 如果 $a_{i}>a_{j}$ ，那么说明 $i$ 可以接在 $j$ 后面形成一个更长的上升子序列，此时长度+1 ，也就是说，我们可以写出下面的状态转移方程：

$$
dp[i]=\max_{0\leq j<i,a_{j}<a_{i}}(dp[j]+1)
$$

显然，这个复杂度是 $O(N^2)$ 的，如果要压复杂度的话我们就要进行下一步思考了。
## 2.2 贪心和二分优化
接下来我们优化这个复杂度。

显然我们的目标是使得这个上升子序列尽可能长，那么对应地，我们要“给后面的数留出一定的上升空间”，也就是说，我们要保证这个上升子序列里的数字增长得越慢越好。

我们维护一个数组 $D$ ，定义 $D_{i}$ 是所有长度 $i$ 的上升子序列中末尾元素的最小值，我们要保证的是：
1. $D$ 是一个单调递增数列；
2. 通过确定 $D$ ，使得对应的上升子序列的长度 $\mathrm{len}$ 最长。

于是我们可以考虑下面的操作：
1. 遍历数组中的每一个数 $x$ ，若 $x$ 比 $D$ 中的最后一个数都大，我们就接在末尾，长度+1；
2. 若 $x$ 偏小，那么就在 $D$ 里找到第一个 $\geq x$ 的数将其替换。

对于第一个操作，我们只需要遍历整个序列就可以了；而对于第二个操作，因为我们能保证 $D$ 是一个单调数列，因此显然可以通过二分解决。

故而我们就将这个过程压缩到了 $O(N\log N)$ 。
## 2.3 代码实现
```cpp
for (int x : v) {
    if (D.empty() || x > D.back()) D.push_back(x);
    else {
        int idx = lower_bound(D.begin(), D.end(), x) - D.begin(); //二分查找
        D[idx] = D;
    }
}
```

## 2.4 正确性证明
### 2.4.1 单调性证明
不妨用归纳法证明：只有一个元素的时候显然 $D_{1} = a_{1}$ 且序列最长。接下来假设前 $i-1$ 个元素都是递增的，我们考虑第 $i$ 个元素 $D_i$：

1. 如果 $x > D_{\mathrm{len}}$ 那么我们将 $x$ 也放在序列末尾，此时序列长度 +1， $D_{\mathrm{len}+1}=x$ ，并且这个新的序列依然是递增的。
2. 如果 $x \leq D_{\mathrm{len}}$ 那么我们在 $d$ 中找到第一个 $\geq x$ 的位置 $j$ ，将 $D_{j}$ 更新为 $x$ ，因为我们保证 $D_{j-1}<x<D_{j+1}$ ，因此我们依旧能够保证这个序列是递增的。

于是我们就能证明在这个策略之下，序列一定是递增的。
### 2.4.2 最优性证明
设 $D_{j}=v$ ，接下来我们证明把 $v$ 替换为 $x$ 不会丢失最优解：

考虑任意一个在 $x$ 之后的元素 $y$ ，如果 $y$ 能接在 $v$ 后面，那么这 $y$ 也一样能接在 $x$ 后面，说明替换并不会使长度变小。

反之如果 $x<y\leq v$ ，说明 $y$ 原本接不上 $v$ 但是能接上 $x$ ，所以反而有扩大序列长度的可能性，故而我们的贪心策略是正确的。
