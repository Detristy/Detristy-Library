# 1. 线索二叉树
## 1.1 二叉树的局限性
我们考虑一棵二叉树，对于树上的每一个结点，其都有两个指针域，一个指向左儿子，一个指向右儿子。
但是对于一些没有左节点或者右节点的中间结点，以及叶子节点而言，至少有一个指针域是空指针，在一棵结点数量为 $n$ 的二叉树上，这样的空指针个数为：

$$
2n-(n-1)n+1
$$

## 1.2 线索树的引入
线索二叉树考虑将这样的空指针利用起来，防止其闲置，大致的利用方法如下：

1. 若该结点的左节点为空，则让它指向该结点的前驱。
2. 若该结点的右结点为空，则让它指向该结点的后继。

我们称这些被“改造”的指针为线索。
那么此时我们就把这些空指针都利用上了，不过为了将线索和指向孩子的指针相区分，我们对于左右指针分别引入一个标志位 `ltag` 和 `rtag`。

当 tag 值为 1 的时候说明其是一个线索；
当 tag 值为 0 的时候说明其指向自己的孩子。

# 2. 线索二叉树的构建
## 2.1 线索二叉树的结点
结点由五部分组成：存储的值，左右指针，左右标志位。
```cpp
typedef struct ThreadNode {  
    int data;  
    ThreadNode * left, * right;  
    bool ltag,rtag;  
    ThreadNode(int val):data(val),left(NULL),right(NULL),ltag(0),rtag(0){}  
}TN;
```
## 2.2 中序线索二叉树的构造
对于中序线索二叉树的构造，我们按照 [[102-1 树#2.2.2 中序遍历 | 中序遍历]] 的顺序，记录下上一个访问结束的节点，我们记录这个节点为 pre 。
在遍历过程中，当指针 p 在某一个节点上的时候，pre 就是 p 的前驱：
1. 此时如果 p 的左指针为空，则让 `p->left = pre` 。
2. 此时如果 pre 的右指针为空，则让 `pre->right = p` 。

注意因为 pre 是根据遍历进行变化的，我们要么将其设为全局变量，要么带入递归函数的参数里，我们在这里将 pre 视作全局变量。

```cpp
typedef struct ThreadNode {  
    int data;  
    ThreadNode * left, * right;  
    bool ltag,rtag;  
    ThreadNode(int val):data(val),left(NULL),right(NULL),ltag(0),rtag(0){}  
}TN;  
  
TN * pre = NULL;  
  
void inThread(TN * p) {  
    if (p == NULL) return;  
    if (p -> ltag == 0) inThread(p->left);  
    if (p -> left == NULL) {  
        p->left = pre;  
        p->ltag = 1;  
    }  
    if (pre != NULL && pre->right == NULL) {  
        pre->right = p;  
        pre->rtag = 1;  
    }  
    pre = p;  
    if (p->rtag == 0) inThread(p->right);  
}  
  
void createInThread(TN * root) {  
    pre = NULL;  
    if (root != NULL) {  
        inThread(root);  
        if (pre->right == NULL) pre->rtag = 1;  
    }  
}
```
## 2.3 先序线索二叉树的构造
在[[102-1 树#2.2.1 先序遍历 | 先序遍历]]中，我们先遍历根节点，再遍历左右节点，此时对于当前节点 p ：
1. 若 `p->ltag == 0` ，则其后继一定是它的左孩子。
2. 若 `p->rtag == 1` ，则访问右指针，此时右孩子和右线索，都是 `p->right` 。

```cpp
typedef struct ThreadNode {  
    int data;  
    ThreadNode * left, * right;  
    bool ltag,rtag;  
    ThreadNode(int val):data(val),left(NULL),right(NULL),ltag(0),rtag(0){}  
}TN;  
  
TN * pre = NULL;  
  
void preThread(TN * p) {  
    if (p == NULL) return;  
    if (p -> ltag == NULL) {  
        p->left = pre;  
        p->ltag = 1;  
    }  
    if (pre != NULL && pre -> right == NULL) {  
        pre->right = p;  
        pre->rtag = 1;  
    }  
    pre = p;  
    if (p->ltag == 0) {  
        preThread(p->left);  
    }  
    if (p->rtag == 0) {  
        preThread(p->right);  
    }  
}  
  
void createPreThread(TN * root) {  
    pre = NULL;  
    if (root != NULL) {  
        preThread(root);  
        if (pre->left == NULL) pre->ltag = 1;  
    }  
}
```
## 2.4 后序线索二叉树的构造
根据 [[102-1 树#2.2.3 后序遍历 | 后序遍历]] ，我们的顺序是先左右子树再访问根。
后序遍历构造线索二叉树既有其优势也有其劣势：

1. 优势：先走左右子树最后才访问根节点，因此修改 `p->left` 的时候，左子树已经遍历结束，此时不会产生无限递归。
2. 劣势：当我们在后序方向寻找 $u$ 的后继的时候，显然 $u$ 的右孩子的后继是 $u$ 本身，但是当 $u$ 的右孩子 $v$ 的右指针 `v->right != NULL` 的时候，我们不能将这个右孩子改成右线索。因此我们需要单独开辟一个新的指针域 `parent` ，此时就需要引入三叉链表。

此时节点结构变为：

```cpp
typedef struct ThreadNode {  
    int data;  
    ThreadNode * left, * right, * parent;  
    bool ltag,rtag;  
    ThreadNode(int val):data(val),left(NULL),right(NULL),parent(NULL),ltag(0),rtag(0){}  
}TN;
```

总体构建变为：

```cpp
struct PostThreadNode {
    int val;
    PostThreadNode *left, *right, *parent;
    int ltag, rtag;
    PostThreadNode(int x) : val(x), left(NULL), right(NULL), parent(NULL), ltag(0), rtag(0) {}
};

PostThreadNode *post_pre = NULL;

// 核心：后序线索化（完全不需防死循环！）
void postThread(PostThreadNode *p) {
    if (p == NULL) return;

    postThread(p->left);
    postThread(p->right);

    // 回溯时最后处理根节点，非常安全
    if (p->left == NULL) {
        p->left = post_pre;
        p->ltag = 1;
    }
    if (post_pre != NULL && post_pre->right == NULL) {
        post_pre->right = p;
        post_pre->rtag = 1;
    }
    post_pre = p;
}
```
