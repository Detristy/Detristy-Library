# 1. 概念
>[!definition] 定义（十字链表）
> 十字链表是稀疏矩阵的一种**链式存储结构**。在十字链表中，每个非零元素节点同时链接在它所在的行链表和列链表中，形成十字交叉的链式结构。当矩阵的非零元素个数和位置在运算过程中频繁变化（如矩阵加法、乘法）时，十字链表比三元组顺序表更加灵活，便于插入和删除操作。

# 2. 存储结构

## 2.1 节点定义
十字链表的每个节点包含五个域：
- `i, j`：行标、列标
- `e`：元素值
- `right`：指向同行中下一个非零元素节点
- `down`：指向同列中下一个非零元素节点

```c
typedef struct OLNode {
    int i, j;              // 行标、列标（从 1 开始）
    ElemType e;            // 元素值
    struct OLNode *right;  // 指向同行下一个节点
    struct OLNode *down;   // 指向同列下一个节点
} OLNode, *OLink;
```

## 2.2 链表头结构
整个十字链表包含两个行/列头指针数组和三个规模参数：

```c
typedef struct {
    OLink *rhead;  // 行头指针数组（rhead[i] 指向第 i 行第一个非零节点）
    OLink *chead;  // 列头指针数组（chead[j] 指向第 j 列第一个非零节点）
    int mu;        // 行数
    int nu;        // 列数
    int tu;        // 非零元素个数
} CrossList;
```

# 3. 基本操作

## 3.1 创建十字链表

```c
void CreateCrossList(CrossList *M, int m, int n, int t) {
    M->mu = m; M->nu = n; M->tu = t;
    
    // 分配行/列头指针数组
    M->rhead = (OLink *)malloc((m + 1) * sizeof(OLink));
    M->chead = (OLink *)malloc((n + 1) * sizeof(OLink));
    for (int i = 1; i <= m; i++) M->rhead[i] = NULL;
    for (int j = 1; j <= n; j++) M->chead[j] = NULL;
    
    // 逐个输入三元组并插入
    for (int k = 0; k < t; k++) {
        int i, j; ElemType e;
        scanf("%d %d %d", &i, &j, &e);
        
        OLNode *p = (OLNode *)malloc(sizeof(OLNode));
        p->i = i; p->j = j; p->e = e;
        
        // 插入行链表
        if (M->rhead[i] == NULL || M->rhead[i]->j > j) {
            p->right = M->rhead[i];
            M->rhead[i] = p;
        } else {
            OLNode *q = M->rhead[i];
            while (q->right && q->right->j < j) q = q->right;
            p->right = q->right;
            q->right = p;
        }
        
        // 插入列链表
        if (M->chead[j] == NULL || M->chead[j]->i > i) {
            p->down = M->chead[j];
            M->chead[j] = p;
        } else {
            OLNode *q = M->chead[j];
            while (q->down && q->down->i < i) q = q->down;
            p->down = q->down;
            q->down = p;
        }
    }
}
```

## 3.2 稀疏矩阵加法

设 $C = A + B$，$A$ 和 $B$ 均为 $m\times n$ 的稀疏矩阵（用十字链表存储），按行逐列扫描，将同行号且同列号的元素相加；若结果非零则插入 $C$，若为零则跳过。

```c
CrossList *AddCrossList(CrossList *A, CrossList *B) {
    CrossList *C = (CrossList *)malloc(sizeof(CrossList));
    C->mu = A->mu; C->nu = A->nu; C->tu = 0;
    
    C->rhead = (OLink *)malloc((C->mu + 1) * sizeof(OLink));
    C->chead = (OLink *)malloc((C->nu + 1) * sizeof(OLink));
    for (int i = 1; i <= C->mu; i++) C->rhead[i] = NULL;
    for (int j = 1; j <= C->nu; j++) C->chead[j] = NULL;
    
    for (int i = 1; i <= A->mu; i++) {
        OLNode *pa = A->rhead[i];
        OLNode *pb = B->rhead[i];
        OLNode *pre = NULL;
        
        while (pa || pb) {
            int col; ElemType sum;
            if (pa && (!pb || pa->j < pb->j)) {
                col = pa->j; sum = pa->e; pa = pa->right;
            } else if (pb && (!pa || pb->j < pa->j)) {
                col = pb->j; sum = pb->e; pb = pb->right;
            } else { // pa->j == pb->j
                col = pa->j; sum = pa->e + pb->e;
                pa = pa->right; pb = pb->right;
            }
            if (sum != 0) {
                // 插入节点到 C（类似创建的插入过程）
                InsertNode(C, i, col, sum);
            }
        }
    }
    return C;
}
```

# 4. 复杂度分析

| 操作 | 时间复杂度 |
|------|-----------|
| 创建十字链表 | $O(t \times s)$，其中 $t$ 为非零元素个数，$s$ 为行/列链表平均长度 |
| 查找元素 $a_{i,j}$ | $O(s)$，沿行或列链表遍历 |
| 插入/删除元素 | $O(s)$，修改两个链表的指针 |
| 矩阵加法 | $O(\mathrm{tu}_A + \mathrm{tu}_B)$，与两矩阵非零元素之和成正比 |

相比于三元组顺序表，十字链表在**插入和删除非零元素时不需要移动大量数据**，因此在矩阵运算（加法、乘法）中更为高效。但每个节点需要额外的指针空间（`right` 和 `down`），存储开销更大。

# 5. 与其他存储方式的对比

| 特性 | 三元组顺序表 | 行逻辑链接表 | 十字链表 |
|------|-------------|-------------|---------|
| 存储方式 | 顺序 | 顺序+索引 | 链式 |
| 查找效率 | $O(\mathrm{tu})$ | $O(s)$（按行） | $O(s)$ |
| 插入/删除 | 需要移动元素 | 需要移动元素 | $O(s)$，灵活 |
| 内存开销 | 小 | 中（多 `rpos` 数组） | 大（每节点两个指针） |
| 适用场景 | 矩阵创建后不再改变 | 需要快速按行访问 | 矩阵元素频繁变化 |

详细的三元组顺序表和行逻辑链接顺序表见 [[103-5 数组#3.4 稀疏矩阵 | 稀疏矩阵]]。
