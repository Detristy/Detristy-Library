# 1. 单链表的构建

## 1.1 单链表的特点

单链表具有如下特点：
1. 元素在内存中是不连续的。
2. 强制顺序访问，链式存储结构是不支持随机访问的。
3. 内存是动态分配的，除非内存用尽，否则空间不会溢出。
4. 单链表只能单向遍历，遍历的复杂度是 $O(n)$ 。

## 1.2 结点

单链表的结点结构由数据域和指针域 next 构成，定义如下：

```cpp
struct Node {  
    int num = 0,score = 0;  
    Node *next;  
    Node() {  
        next = NULL;  
    }  
};
```

单链表通常由头节点开始，有的也可以不带头节点，头节点通常不存储有效数据，其指针域 next 指向首元结点，称之为哑头结点。
运用头节点的优势是便于头插和删除第一个元素，同时对于链表头插并不需要特判。

> [!faq] 头指针和头节点
> 头指针和头节点是两个不同的概念。
> 带有头节点的链表中，头指针 head 通常指向头结点。
> 不带头节点的链表中，头指针通常直接指向首元节点。

> [!warning] 易错 1
> 所有的链表都需要通过头指针作为“入口”。


# 2. 单链表操作

## 2.1 遍历和查找

新建立一个指针从头指针（或首元结点）开始向前遍历，每一次让将 next 指针域赋值给自己。

```cpp
void Traversal() {  
    if (head == NULL) return;  
    Node* p = head;  
    while (p->next != NULL) {  
        p = p->next;  
    }  
}
```

查找的操作，在遍历的基础上添加查找位置 pos ，遍历的时候计数即可。

> [!faq] 位置 pos 的合法性
> 在两种情况下插入位置是不合法的：
> 1. pos $<1$ 时不合法，这个可以在一开始就进行判断。
> 2. pos 大于链表长度时不合法，这个可以在遍历结束判断当前位置是否为空进行判断。

```cpp
void Find(int pos) {  
    if (head == NULL || pos < 1) {  
        cout << "Not Found\n";  
        return;  
    }  
    Node* p = head;  
    int cnt = 0;  
    while (p != NULL && cnt <= pos) {  
        p = p->next;  
        cnt++;  
    }  
    if (p == NULL) {  
        cout << "Not Found\n";  
        return;  
    }  
    cout << p->data;  
}
```

## 2.2 头插

头插即每次将新结点插入到头节点后，即链表的最前端，因此头插后的链表结点顺序和输入顺序是相反的，即倒序建表。

需要说明的是，原本对头节点判空的时候写的是：

```cpp  
if (head == nullptr) return;  
```

这里调整为如果头节点为空则新建一个头节点（下同），完整代码如下：

```cpp
void InsertHead(Elemtype data) {  
    Node *newnode = new Node();  
    newnode->data = data; 
    
    if (head == NULL) {  
        head = new Node();  
        newnode->next = NULL;  
    }  
    else {  
        newnode->next = head->next;  
    }  
    
    head->next = newnode;  
}
```

## 2.3 尾插

因为每一次都要在链表的结尾插入，故每次插入时都要找到尾节点。

```cpp
void InsertTail(Elemtype data) {  
    Node *newnode = new Node();  
    
    newnode->next = NULL;  
    newnode->data = data; 
    
    if (head == NULL) {  
        head = new Node();  
        head->next = newnode;  
    }  
    else {  
        Node *p = head;  
        while (p->next != NULL) {  
            p = p->next;  //找尾节点
        }  
        p->next = newnode;  
    }  
}
```

如果输入数据量过大，每一次尾插都要找尾节点，时间复杂度是 $O(n^2)$ 的，为了快速到达尾节点，我们考虑维护一个尾指针 tail 使其永远指向尾节点。

```cpp
void InsertTail(Elemtype data) {  
    Node *newnode = new Node();  
    newnode->next = NULL;  
    newnode->data = data;  
      
    if (head == NULL) {  
        head = new Node();  
        head->next = newnode;  
        tail = head;  
    }  
    else {  
        tail->next = newnode;  
        tail = tail->next;  
    }  
}
```

## 2.4 特定位置插入

首先对链表进行遍历，通过计数的方式找到目标位置 pos 的前驱结点，接着执行插入操作。

这里我们设定函数类型为 bool ，这样可以返回插入是否成功。

```cpp
bool Insert(int pos,Elemtype data) {  
    if (pos < 1) return false;  
    Node *p = head;  
    int cnt = 0;  
    
    while (p != NULL && cnt < pos-1) {  
        p = p->next;  
        cnt++;  
    }  
    
    if (p == NULL) return false;  
    Node *newnode = new Node();  
    newnode->data = data;  
    newnode->next = p->next;  
    p->next = newnode;  
    return true;  
}
```

## 2.5 链表逆置

这里使用头插法对链表进行逆置，首先将链表和头节点断开，接着从前往后遍历这个链表头插在头节点处。

> [!faq] 尾指针 tail 的处理
> 逆置的过程中 `head->next` 一直在变化，因为头插法会使得原本的首元节点不断向后推移，原本的首元节点在逆置结束之后就是新链表的尾节点，我们把首元结点存下来，在逆置结束之后赋值给尾指针即可。

```cpp
void reverse() {  
    Node *p = head->next;  
    Node *tmp;  
    tail = head->next;  
    head->next = NULL;  
    while (p != NULL) {  
        tmp = p->next;  
        p->next = head->next;  
        head->next = p;  
        p = tmp;  
    }  
}
```

## 2.6 头删

头删即删除头节点后的首元结点。删除前需要判空：链表为空（`head == NULL`）或只有头节点（`head->next == NULL`）时不能删除。

```cpp
bool DeleteHead() {
    if (head == NULL || head->next == NULL) return false;
    
    Node *p = head->next;       // p 指向首元结点
    head->next = p->next;       // 头节点跳过 p
    delete p;                   // 释放 p
    return true;
}
```

> [!warning] 易错
> 删除后如果链表变空，`head->next == NULL`，此时若有尾指针 `tail`，需要将其指向头节点 `tail = head`。

## 2.7 尾删

尾删需要找到尾节点的前驱结点。如果维护了尾指针 `tail`，仍然需要遍历找到 `tail` 的前驱（因为单链表无法从 `tail` 反向回溯）。

```cpp
bool DeleteTail() {
    if (head == NULL || head->next == NULL) return false;
    
    Node *p = head;
    while (p->next->next != NULL) {  // 找到倒数第二个结点
        p = p->next;
    }
    
    delete p->next;      // 释放尾节点
    p->next = NULL;      // p 成为新的尾节点
    
    if (tail != NULL)    // 维护尾指针
        tail = p;
    return true;
}
```

> [!faq] 时间复杂度
> 即使维护了尾指针，尾删仍然需要 $O(n)$ 遍历找前驱。这是单链表的固有缺陷——只能单向访问。[[103-1A 双链表 | 双链表]] 可以 $O(1)$ 实现尾删。

## 2.8 特定位置删除

遍历找到目标位置 `pos` 的前驱结点，然后删除其后继。删除原理和特定位置插入类似，都需要找到前驱。

```cpp
bool Delete(int pos) {
    if (pos < 1 || head == NULL || head->next == NULL) return false;
    
    Node *p = head;
    int cnt = 0;
    
    while (p->next != NULL && cnt < pos - 1) {
        p = p->next;
        cnt++;
    }
    
    if (p->next == NULL) return false;  // pos 超出链表长度
    
    Node *tmp = p->next;      // tmp 指向待删结点
    p->next = tmp->next;      // 前驱跳过 tmp
    delete tmp;               // 释放内存
    return true;
}
```

> [!faq] 位置 pos 的合法性
> 与插入操作相同，两种情况下删除位置不合法：
> 1. `pos < 1` 时不合法。
> 2. `pos` 大于链表长度时不合法——遍历结束后 `p->next == NULL` 即说明 `pos` 越界。

## 2.9 按值删除

遍历链表，找到第一个值为 `target` 的结点并将其删除。由于需要修改前驱的 `next` 指针，遍历时维护一个指向前驱的指针。

```cpp
bool DeleteByValue(Elemtype target) {
    if (head == NULL || head->next == NULL) return false;
    
    Node *prev = head;
    Node *p = head->next;
    
    while (p != NULL) {
        if (p->data == target) {
            prev->next = p->next;   // 前驱跳过 p
            delete p;               // 释放 p
            return true;
        }
        prev = p;
        p = p->next;
    }
    return false;  // 未找到 target
}
```

> [!faq] 删除所有值为 target 的结点
> 将 `return true` 改为 `p = prev->next; continue;`，并在函数末尾返回删除的结点总数即可。

## 2.10 清空链表

释放除头节点外的所有结点，保留头节点以便后续继续使用链表。

```cpp
void Clear() {
    if (head == NULL) return;
    
    Node *p = head->next;
    while (p != NULL) {
        Node *tmp = p;
        p = p->next;
        delete tmp;
    }
    head->next = NULL;
    tail = head;  // 维护尾指针
}
```

> [!warning] 易错
> 释放结点前必须先保存 `p->next`，否则 `delete` 之后无法继续向后遍历。这与遍历时不能「先删再走」是同一个道理。
