# 1. 数组
数组实际上是一块连续的内存，元素在数组内按照某种顺序连续存储。

数组允许 $O(1)$ 的随机访问，比链表效率更高。

# 2. 寻址公式
上面提到数组是一块连续的内存，因此我们可以通过使用寻址公式来快速定位数组内任意位置的元素。

## 2.1 一维数组
设一维数组 $A$ 中每个元素占 $L$ 个字节，首地址为 $\mathrm{Loc}(A[0])$ （这里默认下标起始为 0，即 0-based），那么下标为 $i$ 的元素地址为：
$$
	\mathrm{Loc(A[i])} = \mathrm{Loc(A[0])}+i\times L
$$

## 2.2 二维数组
二维数组相当于一个 $m$ 行 $n$ 列的矩阵，并把这个矩阵塞入了一维内存。

二维数组分为两种：行优先和列优先。行优先就是一行一行按顺序排列，列优先类似。

#### 行优先的寻址公式
行优先的寻址公式是当前位置所在的行乘以列宽在加上当前所在的列数。
$$
	\mathrm{Loc}(A[i][j])=\mathrm{Loc}(A[0][0])+(i\times n+j)\times L
$$
#### 列优先的寻址公式
行优先的寻址公式是当前位置所在的列乘以行宽在加上当前所在的行数。
$$
	\mathrm{Loc}(A[i][j])=\mathrm{Loc}(A[0][0])+(j\times m+i)\times L
$$

# 3. 矩阵的压缩存储
## 3.1 对称矩阵
我们的目标是将对称矩阵存在一个一维数组里，因此我们需要每个元素 $m_{i,j}$ 在一维数组里的下标 $\mathrm{i\mathrm{d}x}(m_{i,j})$

因为矩阵是对称的，因此我们可以只存储一半（上三角或下三角）。

我们接下来只考虑上三角的部分，矩阵有 $n$ 列，对于第 $i$ 行，有 $n-i+1$ 个元素。
对于 $A[i][j]$ ，前 $i$ 行有 $n+(n-1)+\dots+(n-(i-1)+1)=\frac{(i-1)(2n-i+2)}{2}$ 个元素，第 $i$ 行有 $j-i$ 个元素，因此对应的下标为：
$$
\mathrm{i\mathrm{d}x}(m_{i,j})=\frac{(i-1)(2n-i+2)}{2}+(j-i)
$$
>[!warning] 对称变换
>当元素位于下三角的时候，需要注意对坐标进行对称变换。

## 3.2 三角矩阵

### 下三角矩阵
下三角矩阵是指主对角线以上的元素全为相同的常量 $c$（通常为 0）的矩阵。压缩存储时，只需存储下三角（含对角线）的元素，外加一个存储常量 $c$ 的位置。

设矩阵为 $n$ 阶方阵，用一维数组 $B$ 按行优先存储。对于 $i \geq j$ 的元素 $a_{i,j}$（行列下标均从 1 开始），其在一维数组中的下标为：
$$
\mathrm{idx}(a_{i,j}) = \frac{i(i-1)}{2} + j - 1
$$
当 $i < j$ 时，元素为常量 $c$，统一存储在数组 $B$ 的最后一个位置（下标 $\frac{n(n+1)}{2}$）。

### 上三角矩阵
上三角矩阵是指主对角线以下的元素全为相同的常量 $c$ 的矩阵。按行优先压缩存储上三角（含对角线）元素。对于 $i \leq j$ 的元素 $a_{i,j}$（行列下标均从 1 开始），其在一维数组中的下标为：
$$
\mathrm{idx}(a_{i,j}) = \frac{(i-1)(2n-i+2)}{2} + (j-i)
$$
当 $i > j$ 时，元素为常量 $c$，统一存储在数组末尾。

## 3.3 对角矩阵（带状矩阵）

对角矩阵是指所有非零元素集中在以主对角线为中心的带状区域内的矩阵。最常见的为**三对角矩阵**，即非零元素仅出现在主对角线及其相邻的上下两条对角线上（满足 $|i-j| \leq 1$），其余位置均为 0。

设 $n$ 阶三对角矩阵按行优先压缩存储到一维数组 $B$ 中。对于带状区域内的元素 $a_{i,j}$（行列下标均从 1 开始），其在一维数组中的下标为：
$$
\mathrm{idx}(a_{i,j}) = 2i + j - 3
$$
带状区域外的元素均为 0，无需存储。一维数组共需 $3n-2$ 个存储单元。

## 3.4 三元表下表示的稀疏矩阵

稀疏矩阵的元素个数 $t$ 远小于矩阵的总大小 $m\times n$，且非零元素无分布规律。如果直接用一维数组压缩存储会浪费很多空间。

### 3.4.1 三元组顺序表
我们可以对每个非零元素分配一个三元组，分别存储其行标、列标和元素的值。
```cpp
#define MAXSIZE 10000

struct Triple{
    int i, j, data;
};

struct TSMatrix {
    int rows, cols, count; //矩阵的行数列数和非零元素个数
    Triple data[MAXSIZE+1]; 
};
```
保证三元组顺序表中的矩阵元素是按行优先、同行内列优先排好序的。
### 3.4.2 稀疏矩阵的转置
#### 列扫描法
显然我们知道原矩阵的行和列就是转置后矩阵的列和行。那么我们不妨按列扫描三元组表，对每个列号依次查找并交换行列存在新的转置后的矩阵内即可。

我们从原矩阵的第 1 列开始，一直扫描到最后一列。对于每一列，都把整个三元组表完整遍历一遍，只要发现列号对得上，就把它交换行列存入新表。

时间复杂度是 $O(nm)$ ，$n$ 是列数，$m$ 是非零元素个数。
#### 快速转置法
$O(nm)$ 的复杂度实在是太慢了，我们不妨考虑能不能一次扫描就知道每个元素转置后所在的位置。

先统计每列非零元素个数，确定转置后每行在三元组表中的起始位置，再遍历原三元组表直接放入目标位置。时间复杂度 $O(n+m)$ 。

```cpp
// 1. 统计每一列的元素个数
for (int i = 1; i <= M.count; i++) {
    num[M.data[i].c]++;
}

// 2. 计算每一列在 T 中的起跑线
cpot[1] = 1;
for (int col = 2; col <= M.cols; col++) {
    cpot[col] = cpot[col - 1] + num[col - 1];
}

// 3. 遍历原表，直接丢进对应的坑位
for (int p = 1; p <= M.count; p++) {
    int col = M.data[p].c;  // 列号
    int q = cpot[col];      // 坑位

    T.data[q].r = M.data[p].c; // 放入新表
    T.data[q].c = M.data[p].r;
    T.data[q].v = M.data[p].v;

    cpot[col]++; // 🌟 极其重要：起跑线后移，给下一个让位
}
```
### 3.4.3 稀疏矩阵的加法
假设我们要计算矩阵 $C=A+B$ ，而我们知道三元组顺序表中都是有序的，因此加法操作就能轻松很多，我们不妨考虑用两个指针 $i$ 和 $j$ 分别遍历两个矩阵：

1. 若 $A[i]$ 的位置在 $B[j]$ 之前，那么此时直接把 $A[i]$ 插入到 $C$ 中，$i$ 后移。
2. 若 $A[i]$ 的位置在 $B[j]$ 之后，那么把 $B[j]$ 插入 $C$ ，$j$ 后移。
3. 若两个位置重合，则把它们的数值加起来 `sum = A[i].data + B[j].data` ，若 $sum \neq 0$ ，则把这个新的元素插入 $C$ ，然后 $i,j$ 同时后移。

```cpp
void add_matrix(const TSMatrix &A, const TSMatrix &B, TSMatrix &C) {  
    C.rows = A.rows; 
    C.cols = A.cols; 
    C.count = 0;  
    
    int i = 1, j = 1;  
  
    while (i <= A.count && j <= B.count) {  
        long long posA = (long long)A.data[i].r * A.cols + A.data[i].c;  
        long long posB = (long long)B.data[j].r * B.cols + B.data[j].c;  
  
        if (posA < posB) {  
            C.data[++C.count] = A.data[i++];  
        } 
        else if (posA > posB) {  
            C.data[++C.count] = B.data[j++];  
        } 
        else { // 坐标重合，数值相加  
            int sum = A.data[i].v + B.data[j].v;  
            if (sum != 0) { // 如果抵消了就不写入  
                C.data[++C.count] = {A.data[i].r, A.data[i].c, sum};  
            }  
            i++;
            j++;  
        }  
    }  
    while (i <= A.count) C.data[++C.count] = A.data[i++];  
    while (j <= B.count) C.data[++C.count] = B.data[j++];  
}
```

减法本质上是加法的逆元，因此是同理的。
### 3.4.4 稀疏矩阵的乘法
考虑计算矩阵 $C = A\times B$ ，优先考虑公式：
$$
	C_{i,j}=\sum A_{i,k}·B_{k,j}
$$
但是这样操作很慢，我们考虑优化，具体的算法是行累加法。

不妨考虑按行处理 $A$ ，对于 $A$ 中的一行 $i$ ：
1. 准备一个累加数组 `tmp[]`
2. 遍历 $A$ 中第 $i$ 行的所有非零元素 $A[p]=(i,k,data_{A})$ 
3. 将 $data_{A}$ 和 $B$ 中第 $k$ 行的所有元素相乘。
4. 把 `tmp` 中的所有数据按照列号顺序，写入 $C$ 的三元组中，清空 `tmp` ，处理下一行。

```cpp
void multiply_matrix(const TSMatrix &A, const TSMatrix &B, TSMatrix &C) {
    if (A.cols != B.rows) return; // 矩阵乘法前提
    C.rows = A.rows; C.cols = B.cols; C.count = 0;

    // 1. 计算 B 矩阵的 rpos (每一行的起点位置)
    vector<int> num(B.rows + 2, 0);
    vector<int> rpos(B.rows + 2, 0);
    for (int i = 1; i <= B.count; i++) num[B.data[i].r]++;
    rpos[1] = 1;
    for (int row = 2; col <= B.rows + 1; row++) { // 预留多一位防越界
        rpos[row] = rpos[row - 1] + num[row - 1];
    }

    // A_rpos 统计 A 矩阵中每一行的起点
    vector<int> A_num(A.rows + 2, 0);
    vector<int> A_rpos(A.rows + 2, 0);
    for (int i = 1; i <= A.count; i++) A_num[A.data[i].r]++;
    A_rpos[1] = 1;
    for (int row = 2; row <= A.rows + 1; row++) {
        A_rpos[row] = A_rpos[row - 1] + A_num[row - 1];
    }

    vector<int> temp(B.cols + 1, 0); // 行累加器

    // 2. 按行开始乘法
    for (int i = 1; i <= A.rows; i++) {
        fill(temp.begin(), temp.end(), 0); // 清空累加器
        C_rpos_current_start = C.count + 1;

        int a_start = A_rpos[i], a_end = A_rpos[i + 1];
        // 遍历 A 的当前行 i 里的所有元素
        for (int p = a_start; p < a_end; p++) {
            int k = A.data[p].c;      // 列号 k
            int v_A = A.data[p].v;    // 权值 v_A

            // 瞬间定位到 B 矩阵的第 k 行范围
            int b_start = rpos[k], b_end = rpos[k + 1];
            for (int q = b_start; q < b_end; q++) {
                int j = B.data[q].c;  // 列号 j
                temp[j] += v_A * B.data[q].v; // 累加到临时行
            }
        }

        // 3. 将累加器中的非零结果写入新表，天然保证了列序递增！
        for (int j = 1; j <= B.cols; j++) {
            if (temp[j] != 0) {
                C.data[++C.count] = {i, j, temp[j]};
            }
        }
    }
}
```
