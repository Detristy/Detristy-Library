# 1. 基数排序 $\mathbf{RADIX\;SORT}$

基数排序也是一种**非比较排序**，它将整数按位数切割成不同的数字，然后从最低位（LSD，Least Significant Digit）到最高位（MSD）依次进行排序。

#### 基本思想
对于一个 $d$ 位十进制整数，基数排序的执行过程如下：

1. 先按**个位**排序（使用计数排序作为子程序）。
2. 再按**十位**排序。
3. 依此类推，直到**最高位**。

每一趟排序必须是**稳定的**（否则会破坏前一趟已经排好的低位顺序）。这也是为什么基数排序通常用计数排序来实现每一趟——计数排序天然稳定。

> 为什么从低位到高位？可以想象卡片排序：如果先按高位排再按低位排，最终结果由最后一次排序（低位）主导，高位排序会被打乱。从低位到高位的顺序保证了最终结果由最高位主导，低位起到"打破平局"的作用。

#### 代码实现

```cpp
// 基数排序（LSD），排序 a[1..n]，所有元素均为非负整数
void radix_sort(int a[], int n) {
    // 找到最大值，确定需要排序的位数
    int max_val = a[1];
    for (int i = 2; i <= n; i++)
        if (a[i] > max_val) max_val = a[i];
    
    int *cnt = new int[10]();       // 十进制，0~9 十个桶
    int *output = new int[n + 1]();
    
    // 对每一位进行计数排序
    // exp = 1, 10, 100, ... 分别对应个位、十位、百位
    for (int exp = 1; max_val / exp > 0; exp *= 10) {
        // 清零计数器
        for (int i = 0; i < 10; i++) cnt[i] = 0;
        
        // 统计当前位每个数字的出现次数
        for (int i = 1; i <= n; i++) {
            int digit = (a[i] / exp) % 10;
            cnt[digit]++;
        }
        
        // 前缀和
        for (int i = 1; i < 10; i++) cnt[i] += cnt[i - 1];
        
        // 从后往前放置（保持稳定性）
        for (int i = n; i >= 1; i--) {
            int digit = (a[i] / exp) % 10;
            output[cnt[digit]] = a[i];
            cnt[digit]--;
        }
        
        // 写回原数组，准备下一趟排序
        for (int i = 1; i <= n; i++) a[i] = output[i];
    }
    
    delete[] cnt;
    delete[] output;
}
```

> **基数可以不是 10**。如果以 $2^8 = 256$ 为基数（按字节切分），可以显著减少趟数，适合对 32 位整数排序（只需 4 趟）。常用基数有 $10$、$16$、$256$、$65536$。

# 2. 性能分析

设元素个数为 $n$，数字的位数为 $d$（如 32 位整数以 10 为基数时 $d \approx 10$，以 256 为基数时 $d = 4$），基数为 $r$。

每趟计数排序的复杂度为 $O(n + r)$，共执行 $d$ 趟。

#### 时间复杂度
$O(d \cdot (n + r))$。

当 $d$ 为常数且 $r$ 与 $n$ 同阶或更小时，时间复杂度可视为 $O(n)$。

#### 空间复杂度
需要计数数组 `cnt[r]` 和输出数组 `output[n+1]`，空间复杂度为 $O(n + r)$。

# 3. 稳定性

每一趟使用的计数排序是稳定的，且低位排序的结果不会被高位排序破坏（高位相等时，低位自然保持相对顺序），因此基数排序整体是**稳定**的。

# 4. 基数排序 vs 比较排序

| 特性 | 基数排序 | 比较排序（快排/归并） |
|------|---------|---------------------|
| 时间复杂度 | $O(d \cdot (n+r))$ | $O(n\log n)$ |
| 是否比较 | 否 | 是 |
| 额外空间 | $O(n+r)$ | 快排 $O(\log n)$ / 归并 $O(n)$ |
| 适用场景 | 整数、定长字符串 | 任意可比较类型 |
| 稳定性 | 稳定 | 取决于算法 |

当 $n$ 很大而 $d$ 很小时（如排 10 万个 32 位整数，4 趟足够），基数排序在理论上优于 $O(n\log n)$ 的比较排序。但在实际应用中，由于缓存局部性等因素，基数排序不一定比快速排序快，需要根据具体场景选择。
