# 1. 希尔排序 $\mathbf{SHELL\;SORT}$
我们知道 [[108-1-1 插入排序 | 插入排序]] 适用于元素趋于有序的情形，希尔排序通过将元素先调整至趋于有序的状态，再使用插入排序，这样可以大大优化时间复杂度。

希尔排序的步骤如下：

1. 将排序序列分为若干子序列，分割为 $L[i,i+d,i+2d,\dots,i+kd]$ ，$d$ 是每个元素间的间隔。
2. 对这些子序列进行插入排序。
3. 减小 $d$ 的值，重复 2 和 3 步骤直到 $d=1$ .

此时整张表已经处于了趋于有序的状态，对整体再使用插入排序即可。
![[希尔排序过程图.png]]

#### 代码实现
```cpp
void ShellSort(int arr[], int size)
{
    for (int gap = size / 2; gap > 0; gap /= 2)
    {
        for (int i = gap; i < size; i++)
        {
            int val = arr[i];
            int j = i - gap;

            while (j >= 0 && arr[j] > val)
            {
                arr[j + gap] = arr[j];
                j -= gap;
            }

            arr[j + gap] = val;
        }
    }
}
```

# 2. 性能分析
希尔排序的复杂度分析较为复杂。
#### 最优情况
在最优情况下，其时间复杂度为 $O(n)$ 。
#### 最坏情况
在最差情况下，其时间复杂度为 $O(n^2)$ 。
#### 一般情况
希尔排序的时间复杂度跟 $n$ 的选取以及 $d$ 的选取有关。
当 $d=2^k-1$ 时，其时间复杂度为 $O(n^{1.5})$ 。
当 $d=2^p\times 3^q\;(p,q\in \mathbf{N})$ 时，时间复杂度为 $O(n\log^2n)$ 。
在 $n$ 在某个特定区间内时，其时间复杂度约为 $O(n^{1.3})$ 。

# 3. 稳定性
希尔排序是一种不稳定的排序算法，因为子表的划分可能会改变相同元素的相对位置。
