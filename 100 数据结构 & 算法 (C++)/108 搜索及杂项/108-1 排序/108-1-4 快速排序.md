# 1. 快速排序 $\mathbf{QUICK\;SORT}$

快速排序的基本思想是**分治**：

1. 在待排序序列中选取一个元素作为**基准**（pivot）。
2. 通过一趟排序将序列划分为两部分：左半部分所有元素 $\leq$ 基准，右半部分所有元素 $\geq$ 基准。
3. 递归地对左右两部分分别进行快速排序，直到子序列长度为 $0$ 或 $1$ 时自然有序。

这个过程称为**划分**（partition）。快速排序的平均性能非常优越，因此得名"快速"。

#### 代码实现

```cpp
void quick_sort(int a[], int l, int r) {
    if (l >= r) return;
    // 选取中间元素作为基准
    int i = l - 1, j = r + 1, x = a[(l + r) >> 1];
    while (i < j) {
        do i++; while (a[i] < x);  // 从左找到第一个 ≥ x 的元素
        do j--; while (a[j] > x);  // 从右找到第一个 ≤ x 的元素
        if (i < j) swap(a[i], a[j]);
    }
    // 此时区间被划分为 [l, j] 和 [j+1, r]
    quick_sort(a, l, j);
    quick_sort(a, j + 1, r);
}
```

> **注意**：递归边界不能写成 `i`，因为 `i` 可能等于 `j` 也可能等于 `j+1`，用 `j` 作分界是安全的。

# 2. 性能分析

#### 最好情况
每次划分都将序列均匀地分成两个长度接近的子序列。此时递归深度为 $\log n$，每层处理的元素总数为 $n$，时间复杂度为 $O(n\log n)$。

#### 最坏情况
每次划分都极不均匀，例如序列已经有序且每次选取第一个元素为基准时，递归深度退化为 $n$，时间复杂度为 $O(n^2)$。

实际使用中通常通过**随机选取基准**或**三数取中**来避免最坏情况。

#### 平均情况
平均时间复杂度为 $O(n\log n)$，且常数因子较小，实际运行速度在众多 $O(n\log n)$ 算法中通常是最快的。

#### 空间复杂度
快速排序使用递归实现，递归栈的深度在最好情况下为 $O(\log n)$，最坏情况下为 $O(n)$。因此空间复杂度为 $O(\log n) \sim O(n)$。

> 快速排序是**原地排序**——除了递归栈外不需要额外的数组空间，所有操作在原数组上通过交换完成。

# 3. 稳定性

快速排序在划分过程中涉及远距离的元素交换（`swap(a[i], a[j])`），会打乱相同元素的相对顺序，因此快速排序是**不稳定**的。
