# 1.插入排序 $\mathbf{INSERTION\;SORT}$

对于一个待排序的表 $L$ ，其可以在 $L(i)$ 处将表分为两部分：

1. 对于前半部分 $L[1,\dots,i-1]$ ，要求是一个有序列。
2. 对于后半部分 $L[i+1,\dots n]$ ，是一个待排序的序列。

我们的目标是将 $L(i)$ 插入前半部分使得其依然有序，步骤如下：

1. 在前半部分找到第一个不小于 $L(i)$ 的位置 $k$。
2. 将 $L[k,..,i-1]$ 的元素后移一位，将 $L(i)$ 插入 $L(k)$ 处。

重复这个过程，直至整张表都有序。

![[直接插入排序过程图.png]]

#### 代码实现
```cpp
void insertion_sort(int arr[], int len) {
  for (int i = 1; i < len; ++i) {
    int key = arr[i];
    int j = i - 1;
    while (j >= 0 && arr[j] > key) {
      arr[j + 1] = arr[j];
      j--;
    }
    arr[j + 1] = key;
  }
}
```

# 2. 性能分析
我们考虑插入排序的时间复杂度跟什么有关。

对于第 $i$ 个元素，我们要向前比较，找到位置后，要对该位置及面元素进行后移。
因此遍历每个元素、查找和移动都对时间复杂度有贡献。
#### 最好情况
在最好的情况下，表中的元素已经有序，此时对于每一个元素只需要比较 1 次，移动 0 个元素。
时间复杂度为 $O(n)$ 。
#### 最坏情况
在最坏的情况下，表中的元素处于逆序，此时对于每一个元素都需要向前比较所有元素，然后将这些元素全部后移。
时间复杂度为 $O(n^2)$ 。

因此我们知道插入排序适用于元素区域有序的情况。

考虑到链式存储结构中，插入元素的时间复杂度是 $O(1)$ 的，因此对于链式存储结构，适合使用插入排序。
# 3. 稳定性
插入排序的过程是先比较再排序的，因此相同元素的相对位置不会发生变化。
从而插入排序是稳定的。

# 4. 优化——折半插入
我们考虑优化向前查找的过程，由于前半部分的序列是有序的，因此我们不妨考虑将顺序查找改为二分查找（需要注意链式存储不能使用）。此时比较元素的次数就被大大减小，比较元素部分的时间复杂度为 $O(n\log n)$ 。

但折半查找不会对后移部分产生优化作用，因此整体的时间复杂度仍然是 $O(n^2)$ 。

#### 折半插入排序的代码实现
元素移动部分使用了memmove函数，但是时间复杂度不变化。
```cpp
void insertion_sort(int arr[], int len) {
  if (len < 2) return;
  for (int i = 1; i != len; ++i) {
    int key = arr[i];
    auto index = upper_bound(arr, arr + i, key) - arr;
    // 使用 memmove 移动元素，比使用 for 循环速度更快，时间复杂度仍为 O(n)
    memmove(arr + index + 1, arr + index, (i - index) * sizeof(int));
    arr[index] = key;
  }
}
```
