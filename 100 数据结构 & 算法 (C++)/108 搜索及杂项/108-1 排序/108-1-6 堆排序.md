# 1. 堆排序 $\mathbf{HEAP\; SORT}$
堆排序基于 [[102-6 堆 | 堆]] 实现。

对于升序排序，我们考虑使用大根堆，此时最大的元素是根节点，我们考虑将这个元素将堆的最后一个元素交换，将有效大小-1，接着调整堆，此时根节点就是第二大的数，让这个数跟倒数第二个元素交换……当这个有效大小坍缩至 1 的时候，我们就完成了升序排序。

降序排序对应地，我们使用小根堆，使用和升序排序同样的操作步骤。

#### 代码实现（降序）
```cpp
#include <iostream>
#include <algorithm>

using namespace std;

const int N = 100010;
int a[N], n;

void down(int u, int heap_size) {
    int t = u; 
    if (u * 2 <= heap_size && a[u * 2] > a[t]) t = u * 2;
    if (u * 2 + 1 <= heap_size && a[u * 2 + 1] > a[t]) t = u * 2 + 1;
    if (u != t) {
        swap(a[u], a[t]);
        down(t, heap_size); // 递归下沉
    }
}

void heap_sort() {
    // 建堆
    for (int i = n / 2; i > 0; i--) {
        down(i, n);
    }

    // 排序
    int cur_size = n;
    while (cur_size > 1) {
        swap(a[1], a[cur_size]);  // 将堆顶的最大值，与当前堆的最后一个元素交换
        cur_size--; // 隔离已经排好序的最大值
        down(1, cur_size); // 让换上去的那个“小不点”下沉，重新把最大值浮到堆顶
    }
}

int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> a[i];
    heap_sort();
    for (int i = 1; i <= n; i++) cout << a[i] << ' ';
    cout << "\n";
    return 0;
}
```

# 2. 性能分析
堆排序的最坏时间复杂度、最好时间复杂度和平均时间复杂度均为 $O(n\log n)$ 。

因为堆排序只在数组内部进行操作，故空间复杂度为 $O(1)$ 。

# 3. 稳定性
堆排序是不稳定的，因为堆排序涉及交换的过程。
