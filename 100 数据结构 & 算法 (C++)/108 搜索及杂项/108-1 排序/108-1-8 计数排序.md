# 1. 计数排序 $\mathbf{COUNTING\;SORT}$

计数排序是一种**非比较排序**——它不通过比较元素之间的大小关系来排序，而是利用元素的值直接确定其位置。

#### 基本思想
1. 统计每个值出现的次数（计数）。
2. 对计数数组求**前缀和**，这样就知道每个值在排序结果中的"终止位置"。
3. 从后往前遍历原数组，根据计数数组将元素放到正确位置（从后往前保证了稳定性）。

计数排序要求元素必须是**整数**且取值范围不能太大。

#### 代码实现

```cpp
// 对数组 a[1..n] 进行计数排序，元素取值范围为 [0, max_val]
void counting_sort(int a[], int n, int max_val) {
    // cnt[i] 表示值为 i 的元素个数
    int *cnt = new int[max_val + 1]();
    int *output = new int[n + 1]();
    
    // 1. 统计每个值的出现次数
    for (int i = 1; i <= n; i++) cnt[a[i]]++;
    
    // 2. 求前缀和：cnt[i] 表示值 ≤ i 的元素个数
    for (int i = 1; i <= max_val; i++) cnt[i] += cnt[i - 1];
    
    // 3. 从后往前放置元素（保证稳定性）
    for (int i = n; i >= 1; i--) {
        output[cnt[a[i]]] = a[i];
        cnt[a[i]]--;
    }
    
    // 4. 写回原数组
    for (int i = 1; i <= n; i++) a[i] = output[i];
    
    delete[] cnt;
    delete[] output;
}
```

> **为什么从后往前遍历？** 假设有两个值相同的元素 A 和 B，A 在原数组中出现在 B 之前。从后往前遍历时，B 会先被放置到更靠后的位置，A 后被放置到更靠前的位置，从而保持了 A 在 B 之前的相对顺序，保证了稳定性。

# 2. 性能分析

设元素个数为 $n$，值域范围为 $k$（即 $[0, k]$）。

#### 时间复杂度
- 统计次数：$O(n)$
- 计算前缀和：$O(k)$
- 放置元素：$O(n)$

总时间复杂度为 $O(n + k)$。

当 $k$ 与 $n$ 同阶时（即值域范围与元素个数相当），计数排序的时间复杂度为 $O(n)$，优于所有基于比较的排序（基于比较的排序下界为 $O(n\log n)$）。

但当 $k \gg n$ 时（如值域很大但元素很少），计数排序的时间和空间开销都会很大，此时不适合使用。

#### 空间复杂度
需要一个计数数组 `cnt[k+1]` 和一个输出数组 `output[n+1]`，空间复杂度为 $O(n + k)$。

# 3. 稳定性

从后往前放置元素的设计保证了计数排序是**稳定**的。

# 4. 适用场景

- 元素为整数且值域范围有限（如成绩排序，分数在 $[0, 100]$）。
- 对小范围整数排序时，效率远高于比较排序。
- 计数排序也是**基数排序**的基础子程序。
