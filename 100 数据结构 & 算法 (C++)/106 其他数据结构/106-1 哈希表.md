# 1. 哈希表
哈希表通过运用哈希函数，将查找表中的关键字映射到一个关键字 $Key$ 对应的地址位置，使得在理想情况下，在散列表中查找的时间复杂度为 $O(1)$ 。

然而，在非理想情况下，可能会出现两个或以上不同关键字通过哈希函数映射到相同地址的情况，我们称之为哈希冲突，此时这些相同地址的关键字被称为同义词。

对于一个哈希表，我们要考察其装满的程度，因此定义 $\alpha$ 用于表示这个值，称之为装填因子，其计算方法为：

$$
\alpha=\frac{n}{m}
$$

其中 $n$ 为表中所存储的键值，$m$ 为表长。

我们接下来讨论有关哈希函数的内容，并且讨论有关哈希冲突及其解决方案的问题。
# 2. 哈希函数
对于不同用途的哈希表，我们应构造对应的哈希函数，在构造时应注意：

1. 函数的定义域必须表中包含全体元素。
2. 函数的值域应尽可能均匀分布，减少冲突。
3. 函数应尽量简单易于计算。

接下来讨论几种哈希函数，我们统一设定 $H(x)$ 是关于 $x$ 的一个哈希函数，$K$ 为键值。
## 2.1 直接定址
直接定址的哈希函数如下：

$$
H(K) = aK+b
$$

可以发现地址和键值呈线性关系，这种方式不会产生冲突。但当键值较为分散时会导致空位较多，导致空间的浪费。
## 2.2 除留余数
我们假设表长为 $m$ ，我们寻找一个接近或等于 $m$ 的质数 $p$ ，使 $K$ 对 $p$ 取模：

$$
H(K)=K\%p
$$

当 $p$ 较为理想的时候，能使得每一个 $K$ 能够等概率的映射到表中的任意位置，减少哈希冲突。
但这种方法不能消除哈希冲突，因为取模函数既不是单射也不是满射。
# 3. 哈希冲突
对于大部分哈希函数，均不能避免哈希冲突，因此我们考虑如何解决哈希冲突，接下来列举两种方法。
## 3.1 拉链法
为了避免同义词发生冲突，我们可以让哈希表中的每一个地址对于一个线性链表，而链表可以随着同义词的增加而不断延长。
![[Pasted image 20260611112320.png]]
拉链法的实现方式类似邻接表。
#### vector实现
```cpp
const int N = 100003; // 找一个近 10 万的质数
vector<int> h[N];

void insert(int x) {
    int k = (x % N + N) % N; // 🌟 负数取模安全锁
    h[k].push_back(x);
}
```
#### 链式前向星
```cpp
const int N = 100003;
int h[N];   // 存储每个桶的链表头指针
int e[N];   // 存储每个节点的具体数值
int ne[N];  // 存储每个节点的 next 指针
int idx;    // 类似前向星的 tot，记录当前用到了第几个节点

void init() {
    memset(h, -1, sizeof(h)); // 🌟 初始化：-1 代表链表为空
    idx = 0;
}

void insert(int x) {
    int k = (x % N + N) % N; // 取模
    
    e[idx] = x;         // 1. 登记数据
    ne[idx] = h[k];     // 2. 新节点指向原本的链表头 (头插法)
    h[k] = idx++;       // 3. 让新节点成为新的“带头大哥”
}
```
## 3.2 开放定址法
开放定值法指的是哈希表的空闲空间既可以被对应地址的关键字占用，也可以被其他非同义词关键字占用。

假设此时哈希函数计算出的地址 $H_{1}$ 已经被其他同义词所占用，我们便可以考虑向后查找下一个位置的地址 $H_{2}$ ，若仍然被占用则继续向前探测直到存在一个地址 $H_{k}$ 使得不发生冲突，则 $H_{k}$ 就是该关键字对应的地址，我们将这个过程称为探测。

设 $H_{i}$ 是第 $i$ 次探测得到的哈希地址，则对应的递推方式为：

$$
H_{i}=(H(K)+d_{i})\%m
$$

其中 $m$ 是表长，$d_{i}$ 是一个增量序列，$i=1,2,3\dots ,k$  $(k \leq m-1)$。
当增量序列确定之后，我们就可以确定四种查找空余位置的方法。
### 3.2.1 线性探测法（线性探测再散列）
这种方法的 $d_{i}$ 是连续的，$d_{i}=1,2,3,\dots,m-1$ 。

即当某个同义词发生冲突之后，考虑一个一个地址向后查找直到找到一个空余位置。
这种探测方法会使得大量元素在相邻的哈希地址上堆积，例如 $i$ 的同义词会挤占 $i+1$ 的位置，而 $i+1$ 此时则要挤占 $i+2$ 的位置……会使得存储和查找效率大大降低。
### 3.2.2 平方探测（二次探测）
这种方法的 $d_{i}=1^2,-1^2,2^2,-2^2,\dots,k^2,-k^2$ ，探测路径像是来回折返进行查找，这种探测方法要求 $m$ 是一个可以被表示为 $4k+3$ 的质数，原因见：[[100Ex-4 平方定址法中表长的选取]]。

这个方法可以有效避免堆积问题，但是无法探测到哈希表中的所有位置，但是至少能探测到一半的位置，适合在装填因子 $\alpha<0.5$ 时使用。
### 3.2.3 双哈希函数法（双散列）
在这种方法中，$d_{i}$ 是一个哈希函数 $i\times H_{2}(K)$ ，也就是说这种方法使用了两个哈希函数，当第一个哈希函数得到的地址产生冲突时，则利用第二个哈希函数计算地址增量，具体形式如下：

$$
H_{i}=(H(K)+iH_{2}(K))\%m
$$

其中 $i$ 是冲突的次数，初始为 $0$ 。
### 3.2.4 伪随机序列法
在这个方法中，$d_{i}$ 实际上是一个伪随机数序列。
# 4. 哈希查找
这部分我们重点讨论在哈希表中查找某个关键字的查找效率。
## 4.1 查找步骤
### 4.1.1 拉链法
直接对对应哈希地址所在的链表进行遍历查找即可，下面给出拉链法的完整代码。
#### vector实现
```cpp
const int N = 100003; // 找一个近 10 万的质数
vector<int> h[N];

void insert(int x) {
    int k = (x % N + N) % N; // 🌟 负数取模安全锁
    h[k].push_back(x);
}

bool find(int x) {
    int k = (x % N + N) % N;
    for (int val : h[k]) {
        if (val == x) return true;
    }
    return false;
}
```
#### 链式前向星
```cpp
const int N = 100003;
int h[N];   // 存储每个桶的链表头指针
int e[N];   // 存储每个节点的具体数值
int ne[N];  // 存储每个节点的 next 指针
int idx;    // 类似前向星的 tot，记录当前用到了第几个节点

void init() {
    memset(h, -1, sizeof(h)); // 🌟 初始化：-1 代表链表为空
    idx = 0;
}

void insert(int x) {
    int k = (x % N + N) % N; // 取模
    
    e[idx] = x;         // 1. 登记数据
    ne[idx] = h[k];     // 2. 新节点指向原本的链表头 (头插法)
    h[k] = idx++;       // 3. 让新节点成为新的“带头大哥”
}

bool find(int x) {
    int k = (x % N + N) % N;
    // 顺着当前桶的链表一直往下找，直到遇到 -1（空指针）
    for (int i = h[k]; i != -1; i = ne[i]) {
        if (e[i] == x) return true;
    }
    return false;
}
```
### 4.1.2 开放定址法
对于一个关键字 $K$ 和给定的哈希函数 $H$ ，我们首先计算其初始地址 $H(K)$ 。

1. 如果当前地址无键值，则查找失败。
2. 如果当前地址有键值：
	1. 与 $K$ 相等，则查找成功。
	2. 与 $K$ 不相等，则用给定的处理方法，根据 $d_{i}$ 计算下一个哈希地址，直到查找成功或失败为止。

下面给出几种方法的代码实现。
#### 线性探测法
```cpp
#include <iostream>
#include <cstring>

using namespace std;

const int N = 200003; 
const int INF = 0x3f3f3f3f; // 用一个不可能出现的极大值代表“空位”

int h[N];

void init() {
    // 初始化所有位置
    fill(h, h + N, INF);
}

// 🌟 核心函数：返回 x 应该待的位置。
// 如果 x 存在，返回它所在的下标；如果不存在，返回它应该被插入的那个“空位”下标
int find_pos(int x) {
    int k = (x % N + N) % N;
    
    // 只要这个位置有人，且不是我自己，就继续往后找
    while (h[k] != INF && h[k] != x) k = (k+1) % N
    return k; 
}

void insert(int x) {
    int k = find_pos(x);
    h[k] = x;
}

bool query(int x) {
    int k = find_pos(x);
    return h[k] == x;
}
```
平方探测法只需在线性探测法的基础上修改增量序列即可。
#### 双哈希函数法
```cpp
#include <iostream>
#include <algorithm>

using namespace std;

const int N = 200003; // 主哈希质数
const int R = 199999; // 辅助哈希质数（比 N 稍小的质数）
const int INF = 0x3f3f3f3f;

int h[N];

void init() {
    fill(h, h + N, INF);
}

// 第一个哈希函数：确定初始位置
int hash1(int x) {
    return (x % N + N) % N;
}

// 第二个哈希函数：确定跳跃步长
// 🌟 核心：确保结果绝对不为 0，且最大不超过 R
int hash2(int x) {
    return R - (x % R + R) % R; 
}

int find_pos(int x) {
    int k = hash1(x);
    int step = hash2(x); // 每个人专属的步长

    // 只要有冲突且不是自己，就按步长不断跳跃
    while (h[k] != INF && h[k] != x) {
        k = (k + step) % N; 
    }
    return k;
}

void insert(int x) {
    int k = find_pos(x);
    h[k] = x;
}

bool query(int x) {
    int k = find_pos(x);
    return h[k] == x;
}
```
# 4.2 查找效率
### 4.2.1 平均查找长度
设对于哈希表 $H$ 中的键值 $K_{i}$ ，使用某种探测方式探测时要比较的次数为 $C_{i}$ ，我们可以得到对于整张哈希表而言，其平均查找长度为：

$$
ASL=\frac{1}{n}\sum_{i=1}^nC_{i}
$$

对于同一组关键字，不同的哈希函数对应不同的哈希表，不同的探测方法也会得到不同的哈希表，同理也对应不同的平均查找长度。
虽然查找效率是 $O(1)$ 的，但是由于冲突的存在，使得其依然需要通过平均查找长度来度量查找效率。
### 4.2.2 影响因素
哈希函数，处理冲突的方法和装填因子有关。
主要依赖于装填因子，直观而言，装填因子越大，冲突可能性越大，反之冲突可能性越小。
