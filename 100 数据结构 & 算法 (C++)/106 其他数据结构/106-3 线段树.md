# 1. 线段树

我们考虑一个区间 $[1,N]$ ，并思考如何用分治的方式去维护它。

不妨考虑二分，用 $mid$ 将其从中间分开，变成两个区间： $[1,mid],[mid+1,N]$，接着我们不断这样二分下去，直到每个区间 $[L,R]$ 都有 $L=R$。
不难发现，这样的二分策略恰好形成了一个二叉树的结构，根节点是 $[1,N]$ ，而对于一个区间 $[L,R]$，我们将其从中点 $mid = \frac{L+R}{2}$ ，处分开，左儿子是 $[L,mid],[mid+1,R]$，直到 $L=R$
这是一颗接近满二叉树的结构，故空间复杂度需要预留 $4N$，为什么呢？

如果 $N=2^k$ ，那么这棵树刚好就是一颗满二叉树，总结点数为：

$$
\text{cnt} = N+\frac{N}{2}+\frac{N}{4}+\cdots+1=2N-1
$$

我们设 $M$ 是大于等于 $N$ 的最小的 2 的幂次，也就是说 $M<2N$ ，那么对于一颗有 $M$ 个叶子节点的二叉树， 总结点数就是 $2M-1$，从而 $2M-1<4N-2$，因此只要我们预留 $4N$ 的空间就不会越界。

现在我们确定了这颗树的结构，于是我们提问，对于一个区间 $[L,R]$，怎么求值呢？我们只需要求这个节点的儿子就可以了，我们用 $S(l,r)$ 代表 $[l,r]$ 的区间和，于是：

$$
S(l,r)=S(l,mid)+S(mid+1,r)
$$

这恰是将值向上传递的过程（pushup）

# 2. 线段树上的操作

## 2.1 线段树上的单点修改
我们考虑最简单的单点修改操作。

对于一棵树，假设我们对 $A[x]$ 加上 $v$ ，我们需要从根节点开始查找，直到锁定 $[x,x]$ 的子节点。
对于线段树而言，是一颗二叉树，所以我们只需要考察 $x$ 和每个节点处的 $mid$ 关系即可，如果 $x\leq mid$ ，我们就走向这个结点的左子树，否则走向右子树，类似于二分，最终我们就能保证在 $l==r$ 的时候，我们所在的节点处就是 $[x,x]$。

而对于一个叶子节点，修改是牵一发而动全身的，因为线段树也是一种维护前缀和的方式，对于一个节点的修改，我们要从下往上遍历修改直到根节点。

原理我们清楚了，但这里依旧存在一个问题，我们怎么找到左子树和右子树呢？

依旧是数学环节，我们设这颗线段树是满的，下标为 1 的位置是根节点，对应这棵树的第 0 层。
接着我们继续推导，第一层有 2 个节点，下标为 2 和 3……，直到第 $D$ 层，就有 $2^D$ 个节点。

于是我们得到一个推论：第 $D$ 层上方所有层级的节点总数之和为：

$$
\sum^{D-1}_{i=0}2^i=2^D-1
$$

这意味着，第 $D$ 层的节点下标是就是从 $2^D$ 开始的。

接着，我们设 $p$ 位于第 $D$ 层，且是这一层从左往右的第 $k$ 个节点 $(k \in [1,2^D])$，那么 $p$ 的下标就是 $2^D+k-1$
我们来考察其左儿子，其位于 $D+1$ 层，那么对于点 $p$ ，其左儿子应该是第 $D+1$ 层的第 $2k-1$ 个节点。

所以我们可以计算其对应的下标 $\text{idx}_l$：

$$
\text{idx}_l = (2^{D+1}-1)+2k-1 = 2(2^D+k-1)
$$

我们将 $p$ 的下标代入得：

$$
\text{idx}_l = 2p
$$

同理，我们可以推出右儿子的节点编号：

$$
\text{idx}_r = 2p+1
$$

这样，我们就找到了左儿子和右儿子，于是我们就能够向下传递了，而这个传递是在树上进行的，因此我们不难证明，单点修改的时间复杂度是 $O(logn)$ 的。

## 2.2 线段树的区间修改

我们当前的任务是给区间 $[L,R]$ 的每个数都加上 $v$ ，如果我们使用单点修改，那么复杂度是 $O(nlogn)$ 的。

我们考虑引入一个叫做 lazy_tag 的标记。

我们考虑区间 $[L,R]$ 和 $[l,r]$，对 $[L,R]$ 每个数都加上 $v$，如果 $[l,r]$ 被 $[L,R]$ 所包含，我们就可以让 $[l,r]$ 整体加上一个值，代表 $[l,r]$ 内所有节点加上 $v$ 的总和：

$$
sum += v*(r-l+1)
$$

这样我们就不需要往下递归了，我们只需要直接获取 $[l,r]$ 更新的值即可。
这个更新的值就是我们要对这个节点打上的 lazy_tag。

那么如果接下来要修改的区间 $[l',r']$ 在 $[l,r]$ 里面呢？

我们只需要将 $[l,r]$ 的 lazy_tag 向儿子传递（pushdown），然后清空自己的 lazy_tag 就可以了。

实现区间修改的时间复杂度也是 $O(\log n)$。

线段树的区间修改操作不需要使用逆运算，只需要满足结合律就可以了，也就是说，他只需要这个代数系统是个半群即可（满足结合律）

此外，我们能够运用线段树来处理符合逻辑，例如我们为 $[L,R]$ 每个数乘以 $k$ ，并且为 $[L,R]$ 每个数加上 $v$ ，我们就可以用两个 lazy_tag 来维护这两个操作进行逻辑分发。

## 2.3 线段树的区间查询
线段树的单点查询较为简单，只需要往下搜索线段树知道对应下标的叶子节点即可，我们主要讨论区间查询：

对于一个在区间 $[L,R]$ 的查询，线段树为我们将其拆成了多个子区间：

假如 $[l,r]$ 被 $[L,R]$ 完全覆盖，那么我们直接返回就可以了。

如果不被完全覆盖，例如 $[1,3]$ 和我们要查询的 $[2,5]$ 有重叠，我们就继续递归：

1. 如果左边界 $L \leq mid$ ，那么我们要查找的区间在左儿子里，我们前往左儿子进一步搜索
2. 如果右边界 $R>mid$ ，那么同理，我们前往右儿子进行进一步搜索

最终我们将这个结果逐步合并至上一层即可。

## 2.4 线段树的构建
我们的目标是将一个数组映射到一个线段树上。

当我们找到根节点 $L==R$ 的时候，此时我们就找到了我们数组里对应的下标 $x$ 到线段树的下标（此时是 $L$ 或 $R$ ）上。

否则我们不断二分，直到到达根节点为止，接着我们需要维护每个儿子的和到父亲：

$$
tree[x] = tree[2x]+tree[2x+1]
$$

```cpp
#include <bits/stdc++.h>
#include <pstl/algorithm_fwd.h>
using namespace std;
typedef long long ll;

int n,m;
const int N = 1e5+10;
ll a[N];

struct segment_tree {
    ll rt[N*4];
    ll lazy[N*4];

    void pushdown(int x,int l,int r) {
        if (!lazy[x]) return;
        int mid = (l+r)/2;
        rt[x*2] += (mid-l+1)*lazy[x];
        rt[x*2+1] += (r-mid)*lazy[x];
        lazy[x*2] += lazy[x];
        lazy[x*2+1] += lazy[x];
        lazy[x] = 0;
    }
    void build(int x = 1,int l = 1,int r = n) {
        if (l == r) {
            rt[x] = a[l];
            return;
        }
        ll mid = (l+r)/2;
        build(x*2,l,mid);
        build(x*2+1,mid+1,r);
        rt[x] = rt[x*2] + rt[x*2+1];
    }
    void mergeP(int pos,ll val,int x = 1,int l = 1,int r = n) {
        if (l == r) {
            rt[x] = val;
            return;
        }
        pushdown(x,l,r);
        ll mid = (l+r)/2;
        if (pos <= mid) mergeP(pos,val,x*2,l,mid);
        else mergeP(pos,val,x*2+1,mid+1,r);
        rt[x] = rt[x*2] + rt[x*2+1];
    }
    void mergeS(int ql,int qr,ll val,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) {
            rt[x] += (r-l+1) * val;
            lazy[x] += val;
            return;
        }
        pushdown(x,l,r);
        ll mid = (l+r)/2;
        if (ql <= mid) mergeS(ql,qr,val,x*2,l,mid);
        if (mid < qr) mergeS(ql,qr,val,x*2+1,mid+1,r);
        rt[x] = rt[x*2] + rt[x*2+1];
    }
    ll query(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) return rt[x];
        pushdown(x,l,r);
        ll mid = (l+r)/2;
        ll ans = 0;
        if (ql <= mid) ans += query(ql,qr,x*2,l,mid);
        if (mid < qr) ans += query(ql,qr,x*2+1,mid+1,r);
        return ans;
    }
}t;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);
    cin >> n >> m;
    for (int i = 1 ; i <= n ; i++) cin >> a[i];
    t.build();
    while (m--) {
        ll op,x,y,k;
        cin >> op >> x >> y;
        if (op == 1) {
            cin >> k;
            t.mergeS(x,y,k);
        }
        else {
            cout << t.query(x,y) << '\n';
        }
    }
}
```

# 3. 线段树的应用
## 3.1 线段树维护多运算

在维护多运算的时候，我们需要关系运算之间的关系，推导对应的 lazy_tag 和 pushup 方法。

见 [[P3373 【模板】线段树2]] 。

## 3.2 线段树维护区间最值
我们将求区间和改为求区间最值，本质上就是将运算的逻辑从加法群切换到了全序集里，很明显，最值运算可以组成半群因为其满足：

$$
max\{max\{a,b\},c\}=max\{a,max\{b,c\}\}
$$

最小值同理，因此这证明了我们使用线段树的正确性。

首先这里只涉及区间查询，因此我们就不需要使用 pushdown 和 lazy_tag 了。

使用线段树来维护区间最值，需要考虑两个问题：区间合并的逻辑 和 pushdown 的逻辑 和 查询的逻辑。

我们先来看区间合并的逻辑，这个很简单，取最值就可以了，我们讨论最大值，最小值是同理的。

设区间 $[l,r]$ 的最大值为 $M(l,r)$ ，于是我们有： $M(l,r)=max\{M(l,mid),M(mid+1,r)\}$ 。
接下来我们来看看如何查询，因为我们既要查最大值也要查最小值，所以我们用两个函数来分别维护，我们继续讨论最大值，最小值是同理的。

需要注意因为最大值和最小值是两个逻辑，因此我们在线段树里需要开两个数组分别维护这两个运算。

下面的代码能够快速求区间内的最大值和最小值的差。

```cpp
#include <bits/stdc++.h>
#include <pstl/algorithm_fwd.h>
using namespace std;
typedef long long ll;

int n,q;
const int N = 5e4+10;
ll h[N];

struct segment_tree {
    ll mx[N*4],mn[N*4];

    void build(int x = 1,int l = 1,int r = n) {
        if (l == r) {
            mx[x] = mn[x] = h[l];
            return;
        }
        ll mid = (l+r)/2;
        build(x*2,l,mid);
        build(x*2+1,mid+1,r);
        mx[x] = max(mx[x*2], mx[x*2+1]);
        mn[x] = min(mn[x*2], mn[x*2+1]);
    }
    ll queryMax(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) return mx[x];
        ll mid = (l+r)/2;
        ll ans = -0x3f3f3f3f;
        if (ql <= mid) ans = max(ans,queryMax(ql,qr,x*2,l,mid));
        if (mid < qr) ans = max(ans,queryMax(ql,qr,x*2+1,mid+1,r));
        return ans;
    }
    ll queryMin(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) return mn[x];
        ll mid = (l+r)/2;
        ll ans = 0x3f3f3f3f;
        if (ql <= mid) ans = min(ans,queryMin(ql,qr,x*2,l,mid));
        if (mid < qr) ans = min(ans,queryMin(ql,qr,x*2+1,mid+1,r));
        return ans;
    }
}t;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);
    cin >> n >> q;
    for (int i = 1 ; i <= n ; i++) cin >> h[i];
    t.build();
    while (q--) {
        int a,b;
        cin >> a >> b;
        cout << t.queryMax(a,b)-t.queryMin(a,b) << "\n";
    }
}

```

## 3.3 线段树维护异或和
很明显，异或运算也是符合结合律的：

$$
(a\oplus b)\oplus c = a\oplus(b\oplus c)
$$

我们的区间修改任务是将 0 修改为 1，将 1 修改为 0，因此我们就不难想到使用异或来解决这个问题，显然：

$$
0\oplus 1=1,1\oplus1=0
$$

因此要进行修改操作，我们只需要为区间里每个数都异或上 1 就可以了。

然而，这题还要求我们求 1 的数量，虽然正常的区间求和可以做到，但是当我们执行异或运算的时候，整个区间的值都会发生变化，因此直接无脑求和显然不行，不妨来研究一下它的性质。

假设在区间 $[l,r]$ 中 1 的个数是 $n$ ，那么当我们执行一次区间内的异或运算的时候，所有的 0 变成 1 ，1 变成 0，那么原本 0 的数量就是现在的 1 的数量，那么此时 1 的数量就是区间长度减去原本 1 的数量： $(l-r+1)-n$ 。

那么pushdown怎么做呢？

我们知道此时的 lazy 应该只有 0 和 1 两种情况，分别代表不反转和反转，我们考虑下面的情况：

1. 此时我们要对一个子区间进行反转，此时这个子区间的 lazy = 0，反转之后明显 lazy = 1 。
2. 此时我们要对一个子区间进行反转，然而 lazy = 1，说明这个区间已经被反转过了，当我们再执行一次反转的时候，就变回了原来的样子，即 lazy = 0 。

因此我们传递 lazy 只需要让下一层的 lazy 异或 1 就可以了，接着照常让这一层的 lazy = 0 即可。

我们用 rt 来维护节点对应的区间内 1 的数量，用 lazy 来标记翻面情况。

```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

int n,m;
const int N = 2e5+10;
int h[N];

struct segment_tree {
    ll rt[N*4];
    int lazy[N*4];

    void pushdown(int x,int l,int r) {
        if (!lazy[x]) return;
        int mid = (l+r)/2;
        //反转后的 1 的数量计算：区间长度-原本1的数量
        rt[x*2] = (mid-l+1) - rt[x*2];
        rt[x*2+1] = (r-mid) - rt[x*2+1];
				//传递 lazy
        lazy[x*2] ^= 1;
        lazy[x*2+1] ^= 1;
        lazy[x] = 0;
    }
    void build(int x = 1,int l = 1,int r = n) {
        if (l == r) {
            rt[x] = h[l];
            return;
        }
        int mid = (l+r)/2;
        build(x*2,l,mid);
        build(x*2+1,mid+1,r);
        rt[x] = rt[x*2] + rt[x*2+1];
    }
    void mergeS(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) {
            rt[x] = (r-l+1) - rt[x];
            lazy[x] ^= 1;
            return;
        }
        pushdown(x,l,r);
        int mid = (l+r)/2;
        if (ql <= mid) mergeS(ql,qr,x*2,l,mid);
        if (qr > mid) mergeS(ql,qr,x*2+1,mid+1,r);
        rt[x] = rt[x*2] + rt[x*2+1];
    }
    ll query(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) return rt[x];
        pushdown(x,l,r);
        int mid = (l+r)/2;
        ll ans = 0;
        if (ql <= mid) ans += query(ql,qr,x*2,l,mid);
        if (qr > mid) ans += query(ql,qr,x*2+1,mid+1,r);
        return ans;
    }
}t;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);
    cin >> n >> m;
    string s; cin >> s;
    for (int i = 0 ; i < n ; i++) h[i+1] = s[i]-'0';
    t.build();
    while (m--) {
        int op,a,b;
        cin >> op >> a >> b;
        if (op) cout << t.query(a,b) << '\n';
        else t.mergeS(a,b);
    }
}

```

## 3.4 线段树维护区间最大公约数
对于 [[107-1 GCD 和 LCM|最大公约数]]，我们可以知道其构成的代数系统是一个半群，即其满足结合律：

$$
\gcd(a,b,c) = \gcd(a,\gcd(b,c))
$$

因此在解决单点修改+区间查询最大公约数的问题时，我们只需要维护每个区间的最大公约数即可。

而区间修改+区间查询最大值的问题则相对要复杂一些，这里我们考虑** [[107-2 裴蜀定理|裴蜀定理]]：**

我们引入一个差分数组 $D$ ，满足 $D_i = A_i - A_{i-1}$ ，那么对于数组里的第 $i$ 个数，其都可以用差分数组进行表示：

$$
A_i= D_1+ D_2+D_3+\cdots+D_i
$$

我们知道： $\gcd(x,y) = \gcd(x,y-x)$
那么对于我们查询的区间最大公约数 $G$：

$$
G=\gcd(A_l,A_{l+1},\cdots,A_{r-1},A_r)
$$

就等于：

$$
G=\gcd(A_l,\gcd(A_{l+1}-A_l,A_{l+2}-A_{l+1},\cdots,A_r-A_{r-1}))
$$

我们将差分数组代入可以继续得到：

$$
G=\gcd(A_l,\gcd(D_{l+1},D_{l+2},\cdots,D_r))
$$

那么这有什么用呢？不难发现当我们对区间内每个数加上 $v$ 的时候，这个区间内的差分值是没有改变的，改变的只是边界处的值和 $A_l$ 。

于是我们只需要用线段树维护一个差分数组就可以了，我们维护这个差分数组的gcd和前缀和即可。

```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

int n,m;
constexpr int N = 5e5+10;
ll a[N],d[N];

struct segmentTree {
    ll t[N*4],g[N*4];
    void build(int x = 1,int l = 1,int r = n) {
        if (l == r) {
            t[x] = d[l];
            g[x] = d[l];
            return;
        }
        int mid = (l+r)/2;
        build(x*2,l,mid);
        build(x*2+1,mid+1,r);
        t[x] = t[x*2] + t[x*2+1];
        g[x] = __gcd(g[x*2],g[x*2+1]);
    }
    void merge(int pos,ll val,int x = 1,int l = 1,int r = n) {
        if (l == r) {
            t[x] += val;
            g[x] += val;
            return;
        }
        int mid = (l+r)/2;
        if (pos <= mid) merge(pos,val,x*2,l,mid) ;
        else merge(pos,val,x*2+1,mid+1,r);
        t[x] = t[x*2] + t[x*2+1];
        g[x] = __gcd(g[x*2],g[x*2+1]);
    }
    ll queryG(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) return g[x];
        int mid = (l+r)/2;
        ll ans = 0;
        if (ql <= mid) ans = queryG(ql,qr,x*2,l,mid);
        if (mid < qr) ans = __gcd(ans,queryG(ql,qr,x*2+1,mid+1,r));
        return ans;
    }
    ll querySum(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) return t[x];
        int mid = (l+r)/2;
        ll ans = 0;
        if (ql <= mid) ans += querySum(ql,qr,x*2,l,mid);
        if (mid < qr) ans += querySum(ql,qr,x*2+1,mid+1,r);
        return ans;
    }
}st;

int main() {
    cin >> n >> m;
    for (int i = 1 ; i <= n ; i++) {
        cin >> a[i];
        d[i] = a[i] - a[i-1];
    }
    st.build();
    while (m--) {
        char op; int l,r;
        cin >> op >> l >> r;
        if (op == 'C') {
```

## 3.5 线段树维护区间平方和
思考如何快速维护区间的方差。

我们知道方差的公式为：

$$
s^2=\frac{1}{n}\sum\limits_{i=1}^n(A_i-\bar A)^2
$$

我们来拆一下这个式子：

$$
s^2=\frac{1}{n}\sum_{i=1}^n(A_i^2-2A_i\bar A+\bar A^2)
$$

$$
s^2 = \frac{1}{n}(\sum_{i=1}^nA_i^2 - 2\sum_{i=1}^nA_i\bar A+\sum_{i=1}^n\bar A^2)
$$

接着我们接着展开：

$$
s^2=\frac{1}{n}\sum_{i=1}^nA_i^2-\bar A^2
$$

也就是说，计算方差需要我们维护一个区间平方和，那么我们如何维护呢？

当我们更新一个区间的时候，其区间平方和的变化如下：

$$
(a_1+x)^2+(a_2+x)^2+\cdots+(a_n+x)^2 = a_1^2 + a_2^2 + \cdots+a_n^2+2x(a_1+a_2+\cdots+a_n)+nx^2
$$

如果我们用 $sq[x]$ 代表节点 $x$ 维护的区间平方和， $sum[x]$ 代表节点 $x$ 维护的区间和， $len$ 代表区间长度，那么我们的pushdown逻辑就有：

$$
sq[2x]=2\times sum[x]\times lazy[x] + len\times (lazy[x])^2
$$

区间修改的逻辑也是同理。接着，方差的公式就可以被转化为：

$$
s^2=\frac{sq}{len}-(\frac{sum}{len})^2
$$

由于 sq 维护的是区间和，那么在 build 的时候，其对应叶子节点的值就是原值的平方，而 pushup 逻辑和 sum 是一样的，单纯求和就可以了。

```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long double ll;

int n,m;
const int N = 1e5+10;
double a[N];

struct segmentTree {
    double t[N*4];
    double lazy[N*4];
    double sq[N*4];

    void pushdown(int x,int l,int r) {
        if (!lazy[x]) return;
        int mid = (l+r)/2;
        //区间平方和的逻辑，记得要写在 t[x] 的更新前面
        sq[x*2] += 2 * t[x*2] * lazy[x] + (mid - l + 1) * lazy[x] * lazy[x];
        sq[x*2+1] += 2 * t[x*2+1] * lazy[x] + (r - mid) * lazy[x] * lazy[x];
        
        t[x*2] += (mid - l + 1) * lazy[x];
        t[x*2+1] += (r-mid) * lazy[x];
        
        lazy[x*2] += lazy[x];
        lazy[x*2+1] += lazy[x];
        lazy[x] = 0;
    }
    void build(int x = 1,int l = 1,int r = n) {
        if (l == r) {
            t[x] = a[l];
            sq[x] = a[l] * a[l]; //sq的初始化
            return;
        }
        int mid = (l + r)/2;
        build(x*2,l,mid);
        build(x*2+1,mid+1,r);
        t[x] = t[x*2] + t[x*2+1];
        sq[x] = sq[x*2] + sq[x*2+1];
    }
    void mergeS(int ql,int qr,double val,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) {
            sq[x] += 2 * val * t[x] + (r - l + 1) * val * val; //区间修改
            t[x] += (r-l+1) * val;
            lazy[x] += val;
            return;
        }
        pushdown(x,l,r);
        int mid = (l + r)/2;
        if (ql <= mid) mergeS(ql,qr,val,x*2,l,mid);
        if (mid < qr) mergeS(ql,qr,val,x*2+1,mid+1,r);
        t[x] = t[x*2] + t[x*2+1];
        sq[x] = sq[x*2] + sq[x*2+1];
    }
    double querySum(int ql,int qr,int x = 1,int l = 1,int r = n) {
        if (ql <= l && r <= qr) {
            return t[x];
        }
        pushdown(x,l,r);
        int mid = (l + r)/2;
        double ans = 0;
        if (ql <= mid) ans += querySum(ql,qr,x*2,l,mid);
        if (mid < qr) ans += querySum(ql,qr,x*2+1,mid+1,r);
        return ans;
    }
    double querySq(int ql,int qr,int x = 1,int l = 1, int r = n) {
        if (ql <= l && r <= qr) return sq[x];
        pushdown(x,l,r);
        int mid = (l + r)/2;
        double ans = 0;
        if (ql <= mid) ans += querySq(ql,qr,x*2,l,mid);
        if (mid < qr) ans += querySq(ql,qr,x*2+1,mid+1,r);
        return ans;
    }
}t;

int main(){
    cin >> n >> m;
    for (int i = 1 ; i <= n ; i++) cin >> a[i];
    t.build();
    while (m--) {
        int op,l,r;
        cin >> op >> l >> r;
        if (op == 1) {
            double k; cin >> k;
            t.mergeS(l,r,k);
        }
        if (op == 2) {
            double len = (double)(r - l + 1);
            cout << fixed << setprecision(4) << t.querySum(l,r) / len << '\n';
        }
        if (op == 3) {
            double len = (double)(r - l + 1);
            double tmp = t.querySum(l,r)/len;
            double sum = t.querySq(l,r)/len - tmp * tmp;
            cout << fixed << setprecision(4) << sum << '\n';
        }
    }
}
```
