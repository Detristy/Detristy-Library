# 「一键关注、终生沦陷」UI 层级结构

> **游戏类型**：虚拟偶像运营 × 牌组构建 × 视觉小说  
> **关键词**：牌组构建、剧情、虚拟现实、视觉、多结局  
> **核心循环**：Hub → 查看活动/社交 → 策划直播 → 编排卡组 → 直播执行 → 结算 → Hub

---

## 一、核心设计理念

游戏有三个「并行系统」需要在 UI 里无缝衔接：

| 系统 | 本质 | UI 隐喻 |
|------|------|---------|
| **牌组构建** | 策略配置 | 「手牌 / 卡册」 |
| **偶像运营** | 资源管理 | 「仪表盘 / 日程表」 |
| **视觉小说** | 叙事推进 | 「对话框 / 场景」 |

**设计原则：Hub 辐条式** —— 一个主界面辐射所有子系统，玩家永远知道「我在哪、下一步去哪」。

## 二、界面层级树

```
┌─ 标题界面 (Title)
│   ├─ 新游戏 → 偶像创建 → 序章教程
│   ├─ 继续游戏 → 主界面 Hub
│   ├─ 画廊 (CG/结局收集)
│   └─ 设置
│
├─ 偶像创建 (Idol Creation)
│   ├─ 原型选择 (赛博歌姬/魔法偶像/近未来主播/宇宙旅者)
│   ├─ 外观定制
│   ├─ 背景故事选择
│   └─ 初始卡组预览
│
├─ ★ 主界面 Hub (核心)
│   ├─ 顶部状态栏 [赛事倒计时 | 粉丝数 | 人气 | 资金 | 偶像体力/心情]
│   ├─ 中央偶像立绘 + 当前状态气泡
│   ├─ 赛事进度条 + 排名速览
│   └─ 底部导航 [直播] [卡组] [活动] [社交] [剧情]
│
│   ├── [直播] Livestream
│   │   ├─ 主题选择 (歌回/杂谈/游戏/联动)
│   │   ├─ 卡牌编排 (拖拽出牌顺序, 限 5-8 张)
│   │   ├─ 时段 & 资源分配
│   │   ├─ ★ 直播执行 (实时模拟: 卡牌触发→观众弹幕→突发事件)
│   │   └─ 结算面板 (数据总结 + 新卡获取 + 粉丝增长)
│   │
│   ├── [卡组] Deck
│   │   ├─ 卡牌图鉴 (按稀有度/类型筛选)
│   │   ├─ 当前卡组编辑器 (30 张上限, 拖拽组卡)
│   │   └─ 卡牌升级/合成 (消耗素材)
│   │
│   ├── [活动] Events
│   │   ├─ 赛事总览 (阶段进度 + 当前排名)
│   │   ├─ 排行榜 (对手排名 + 差距分析)
│   │   └─ 限时活动 (节日/联动/突发事件)
│   │
│   ├── [社交] Social
│   │   ├─ 话题趋势 (热点雷达图 + 蹭热度决策)
│   │   ├─ 粉丝互动 (私信/评论回复)
│   │   └─ 舆情管理 (负面节奏应对, 卡牌选择)
│   │
│   └── [剧情] Story
│       ├─ 主线剧情 (视觉小说式: 立绘+对话框+选择肢)
│       ├─ 个人剧情 (偶像专属故事线)
│       ├─ 支线事件 (触发式短篇)
│       └─ 剧情回顾 / 日志
│
├─ 赛事结算 (Tournament Result)
│   ├─ 最终排名
│   ├─ 结局演出 (根据选择 + 数据走不同结局)
│   └─ 解锁画廊条目
│
└─ 画廊 (Gallery)
    ├─ CG 图鉴
    ├─ 结局列表 (已解锁/未解锁)
    └─ 成就展示
```

## 三、关键界面逻辑

### 3.1 主界面 Hub 的「信息密度控制」

Hub 最容易变成数据轰炸现场。用**三层信息**渐进展示：

| 层级 | 描述 | 交互方式 | 示例 |
|------|------|----------|------|
| **L1 扫一眼就懂** | 偶像立绘 + 赛事倒计时 + 粉丝数大数字 | 顶部栏常驻 | 48:00:00 / 1.2M粉丝 |
| **L2 点一下展开** | 当前排名、体力/心情、待处理事件红点 | 中央区域 | 排名 #12/64, 体力 78% |
| **L3 切 Tab 再看** | 详细卡组、活动列表、社交动态 | 底部导航 | 直播间、卡组编辑器 |

### 3.2 直播执行界面 —— 核心体验层

```
┌────────────────────────────────────┐
│  直播中...          👁 12.3k 观看  │
│  ┌──────────────────────────────┐  │
│  │    偶像 Live2D 立绘动画      │  │  ← 视觉焦点
│  │    (表情随卡牌效果变化)      │  │
│  └──────────────────────────────┘  │
│  ┌──────────────────────────────┐  │
│  │  弹幕流:                     │  │
│  │  "好可爱!!" "唱得真好"       │  │  ← 实时反馈
│  │  "主播看看这个..."           │  │
│  └──────────────────────────────┘  │
│  ┌──────────────────────────────┐  │
│  │ [卡牌手牌区]                 │  │
│  │  🎤高音爆发  💬读弹幕  🎭变装│  │  ← 玩家操作区
│  │  ⚡即兴发挥  🎵点歌  ...     │  │
│  └──────────────────────────────┘  │
│  人气槽: [████████░░] 82%          │
└────────────────────────────────────┘
```

**四层连锁反馈**：  
卡牌打出 → 偶像动作变化 → 弹幕反应 → 人气槽变化

- 突发事件以「弹幕刷屏」或「通知弹窗」打断，玩家需限时应对
- 每场直播 3~5 分钟，保持节奏紧凑

### 3.3 卡组编辑器

牌组构建类游戏，编辑器的体验决定复玩率：

- **拖拽组卡**：从卡池拖入卡组槽位（30 张上限）
- **费用曲线可视化**：右上角显示费用分布柱状图
- **流派标签**：自动检测卡组流派（歌回流/杂谈流/节奏流），显示流派加成
- **快捷操作**：一键清空 / 套用模板 / 自动填充推荐

│         │ (Live2D) │               │

## 四、多结局触发逻辑

结局走向由 **三个维度交叉** 决定：

| 维度 | 来源 | 影响 |
|------|------|------|
| 关键剧情选择 | 视觉小说分支 | 决定大方向 |
| 赛事最终排名 | 运营结果 | 决定成功度 |
| 特定卡牌持有 | 牌组构建达成 | 隐藏路线解锁 |

> 💡 画廊用**树形图**展示分支点：已走路线高亮，未走路线灰显带「?」

## 五、视觉风格

**「赛博玻璃态」**：近未来 × 赛博 × 魔法的混合世界观下的统一视觉语言

- 半透明深色面板 + 霓虹描边（对应不同偶像主题色变化）
- 数据面板用 HUD 风格的细线框 + 数字跳动动画
- 偶像立绘区域保持「干净」，让角色成为视觉重心
- 弹幕流用半透明渐隐效果，不抢主视觉

### 主题色方案

| 偶像原型 | 主色 | 氛围 |
|----------|------|------|
| 赛博歌姬 | `#e94560` 霓虹粉 | 科技感、电音 |
| 魔法偶像 | `#a78bfa` 紫罗兰 | 神秘、魔法阵 |
| 近未来主播 | `#22d3ee` 赛博青 | 都市、直播感 |
| 宇宙旅者 | `#fbbf24` 星尘金 | 广阔、星际 |

## 六、核心游戏循环

```
Hub (主界面)
  │
  ├─→ 查看活动/社交 (获取当前局势情报)
  │     │
  │     └─→ 策划直播策略 (选择主题、时机)
  │           │
  │           └─→ 编排卡组 (策略转化为手牌)
  │                 │
  │                 └─→ 直播执行 ★ (实时模拟 + 卡牌连锁)
  │                       │
  │                       └─→ 结算 (数据总结 + 奖励)
  │                             │
  │                             └─→ Hub (开启下一轮)
  │
  └─→ 剧情事件 (关键节点触发，打断循环，推动叙事)
```

---

## 七、开发优先级建议

1. **P0 — 直播执行界面原型**：整个游戏的高潮体验，四层连锁反馈是最核心的手感
2. **P1 — Hub 主界面 + 底部导航**：所有系统的「锚点」，框架搭好再填内容
3. **P2 — 卡组编辑器**：牌组构建的核心交互，拖拽体验决定复玩率
4. **P3 — 视觉小说系统**：独立开发，集成到 Hub 的剧情 Tab
5. **P4 — 赛事结算 + 画廊**：收束体验，驱动多周目
